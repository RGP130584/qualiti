# ADR-002: HttpOnly Cookies para Sessão JWT

*   **Status**: Aprovada
*   **Data**: 2026-06-08
*   **Autor**: Liderança de Segurança

## Contexto

Previamente, o token JWT gerado pelo backend era armazenado no `localStorage` do frontend Next.js. Essa prática é vulnerável a ataques de roubo de sessão caso ocorra qualquer vulnerabilidade de Cross-Site Scripting (XSS), pois scripts maliciosos injetados poderiam ler diretamente a chave do armazenamento local.

## Decisão

Decidimos migrar o tráfego e armazenamento do token JWT para **HTTP Cookies** com as flags de segurança ativadas:
*   `HttpOnly`: Impede que qualquer script rodando no navegador (`document.cookie`) acesse o token.
*   `Secure`: Garante que o cookie só seja transmitido em conexões criptografadas HTTPS (será configurada como dinâmica para permitir desenvolvimento em localhost via HTTP comum).
*   `SameSite=Strict`: Previne que o cookie seja enviado em requisições de origem cruzada (mitigando ataques de CSRF).

## Consequências

*   **Positivas**:
    *   Mitigação completa de ataques XSS que visam o roubo de tokens de sessão.
    *   Prevenção robusta contra CSRF devido à configuração `SameSite=Strict`.
*   **Negativas**:
    *   Necessidade de refatoração de chamadas API no lado do cliente (o navegador passa a enviar os cookies automaticamente).
    *   Complexidade adicional em requisições SSR no Next.js (o servidor Next.js precisa obter o cookie da requisição recebida e explicitamente encaminhá-lo ao backend Fastify).
