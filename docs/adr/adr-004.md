# ADR-004: Unificação de Persistência no PostgreSQL

*   **Status**: Aprovada
*   **Data**: 2026-06-08
*   **Autor**: Liderança de Dados

## Contexto

O ecossistema vinha sofrendo de redundância de dados devido à coexistência de tabelas duplicadas criadas em diferentes fases do projeto e do módulo de governança antigo. Isso gerava anomalias em atualizações e queries cruzadas desorganizadas.

## Decisão

Unificamos a persistência em um único banco relacional **PostgreSQL** hospedado em container docker. Todas as tabelas duplicadas ou de versões anteriores obsoletas (V1) foram eliminadas ou migradas para os novos esquemas consolidados definidos na inicialização da aplicação (conforme `db.ts`).

A escrita e leitura nas tabelas passa a ser governada de acordo com as fronteiras de propriedade estabelecidas no manual de domínios (Wave 0).

## Consequências

*   **Positivas**:
    *   Eliminação completa de inconsistências de dados por redundância de tabelas.
    *   Esquema limpo e otimizado para operações CRUD e transacionais.
    *   Estrutura relacional consolidada servindo de base sólida para os scanners do TPM.
*   **Negativas**:
    *   Necessidade de migração de dados de instalações antigas para o novo modelo de dados.
