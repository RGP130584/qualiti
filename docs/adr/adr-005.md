# ADR-005: pgvector para Embeddings de RAG

*   **Status**: Aprovada
*   **Data**: 2026-06-08
*   **Autor**: Liderança de Inteligência Artificial

## Contexto

A plataforma necessita realizar buscas semânticas rápidas sobre os documentos, POPs e manuais de acreditação ONA de forma a alimentar o motor de RAG (Retrieval-Augmented Generation) dos agentes de IA. Trazer um banco de dados vetorial externo dedicado (como Pinecone ou Chroma) aumentaria o overhead operacional e o custo de infraestrutura do sistema.

## Decisão

Adotamos a extensão **`pgvector`** integrada diretamente ao banco relacional PostgreSQL do QualitiOS. Isso nos permite armazenar os embeddings de vetores (por exemplo, gerados por modelos de OpenAI ou locais) na mesma base relacional, realizando buscas semânticas (cálculo de distância por cosseno) através de consultas SQL comuns otimizadas com índices vetoriais (HNSW ou IVFFlat).

## Consequências

*   **Positivas**:
    *   Simplicidade de infraestrutura: apenas um container de banco PostgreSQL atende transações de dados e buscas vetoriais.
    *   Consistência transacional ACIDE entre metadados relacionais e embeddings de vetores.
    *   Custo zero de licenciamento de SaaS adicionais de banco vetorial.
*   **Negativas**:
    *   Exige que a imagem docker do PostgreSQL utilizada no deploy tenha a extensão `pgvector` pré-compilada/instalada.
    *   O consumo de memória RAM do banco de dados pode aumentar devido aos índices vetoriais mantidos em cache.
