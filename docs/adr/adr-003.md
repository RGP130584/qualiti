# ADR-003: CORS Fechado por Origens

*   **Status**: Aprovada
*   **Data**: 2026-06-08
*   **Autor**: Liderança de Segurança

## Contexto

A API do backend Fastify estava configurada para aceitar requisições de qualquer origem (`origin: '*'`). Em sistemas de saúde corporativos como o QualitiOS, expor a API sem restrições de origem aumenta o risco de acessos não autorizados de origens cross-origin invasoras.

## Decisão

Optou-se por **desativar o wildcard CORS** e implementar uma política estrita de aceitação de origens. Os domínios permitidos serão cadastrados em uma lista (array) na variável de ambiente `ALLOWED_ORIGINS` no `.env`. O middleware do Fastify e a configuração do Caddyfile devem carregar e rejeitar quaisquer chamadas vindas de origens não autorizadas.

## Consequências

*   **Positivas**:
    *   Prevenção de ataques de requisição cross-origin não autorizada direto à API do backend.
    *   Alinhamento com as regras de governança e auditoria ONA e de segurança da informação hospitalar.
*   **Negativas**:
    *   Exige a correta parametrização de variáveis de ambiente em cada ambiente de deploy (desenvolvimento, staging, produção).
